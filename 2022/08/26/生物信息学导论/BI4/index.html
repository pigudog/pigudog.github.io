From Sequence to NGS

从二十世纪开始人们就将测序，也就是确定以特定DNA分子的序列，作为理解生命的重要方法

但是直到1977年，才开始真正的DNA测序，进入21世纪，以2005年 454 GS-20技术发布为标准，新一代测序技术开始登上舞台。

Read：A short DNA fragment which is read out by sequencer(always in FASTQ format)

  • DNA sequence(symbols)
  • Quality information

Quality: Given p= the probability of a base calling is wrong,its Quality Score can be written as

注意：Q的转化表，是的数值对应了相应的ascii码字符，保存在fastq文件中

  • 在分析过程中，常将错误概率大于0.01，也就是Q小于20的碱基认为是不可靠的
  • 当这样的碱基超过reads的20%，就可以考虑将此条reads丢弃掉


如果使用Paired-end Reads，需要对两条测序结果进行区分

新一代的测序技术还被研究

  • 表观遗传学修饰
  • RNA转录组
  • 以及蛋白DNA相互作用
    等重要的生物学问题

通过将测序得到的DNA测序组，map到reference genome，可以有效地发现不同个体之间存在的遗传差异，这样在通过与表型差异相联系就可以开展探究表型差异的遗传学基础,从而研究遗传学疾病的机制，为后续的诊断和治疗提供重要的线索

  • Association Study:
    • for the given phenotypic trait,”functional varients” could be identified by comparing allele frequencies at hundreds of thousands of polymorphic sites,i.e allele A is associated with phenotypic trait P if (and only if) people who have P also have A more(or less) often than would be predicted from individual frequencies of A and P in the assessed population

RNA-Seq：Explore the transcriptome 通过深度测序研究转录组的技术，通过系统测序细胞中表达的全部转录本

  • “A transcriptome is a collection of all the transcripts present in a given cell”
  • 快速鉴别对应转录组，然后确定对应的剪切体
  • short read is split by intron when aligning to reference Genome
  • 外显子和内含子
    • intron是真核生物细胞DNA中的间插序列。这些序列被转录到前提RNA中，经过剪接被取出，最终不存在成熟的RNA分子中
    • extron（expressed region）在剪接中被保存，它在剪接后仍然会被保存下来，并在蛋白质和成果中表达为蛋白质

ChIP-Seq：Chromatin ImmunoPrecipitation Sequencing

  • 与RNA-Seq相似，深度测序研究RNA转录
  • 但是不同的是：测定与特定抗体结合的DNA序列，并进一步 Profile Protein-DNA interaction
  • 通过选择不同的抗体，既可以研究转录因子的结合位点，也可以用来探测特定的染色质修饰区域

NGS(Next Generation Sequencing): Reads Mapping

Reads Mapping,简单地说就是将测序技术得到的DNA片段，也就是Reads，定位在基因组上

  • Technological: Reads is usual too short to be used/assembled de novo
  • Scientific: Taking full usage of existing annoation/knowledge

Reads Mapping常常作为深度测序分析的第一步，其质量好坏和测序速度都会对后续的分析产生影响

Mapping： Input Data

  • Reference Genome
    • Nucleotide
    • Length:Hundreds of Mb per Chromosome
    • ~3 Gb in total for human genome
  • Reads
    • Nucleotide,with various qualities (relatively high error rate:1e-2~1e-5)
    • Length:36~80 bp per read
    • Hundreds of Gbs per run

“Embedded” Alignment

One sequence is “embedded” in the other sequence(NGS Reads,PCR primer,etc)

What we need here is actually a hybrid “global-local“ alignment

  • “Gobal“ for short sequence (i.e. Read)
  • But “Local“ for long sequence (i.e. Reference Genome)
  • In particular,the surrounding “overhang“ gaps should be not penalized

BLAST Ideas:Seeding-and-extending

  1. Find matches (seed) between the query and subject
  2. Extend seed into High Scoring Segment Pairs(HSPs)
    • Run Smith-Waterman algorithm on the specified region only
  3. Assess the reliability of the alignmet

所谓的索引就是对于数据的划分，可以更好地降低平均的检索时间——Hash

Hash function maps (partial) data into (hashed) keys for following-up indexing,在理想情况下，我们只需要常数时间，O(1)就能完成对于数据的查找

比如说：HBS——A naive hash function
Let’s assume: A=1,C=2,G=4,T=8,then:$HBS(S)=\sum_{i}HBS(S_i)$
like:$HBS(AAAAA) = 1+1+1+1+1=5$

为了提高灵敏度，我们往往会允许若干碱基的错配。这时，我们就可以根据抽屉原理 Pigeonhole principle 将read划分成若干个不重叠的块

  • “In mathematics,the pigeonhole principle states if n items are put into m pigeonholes with n>m,then at least one pigeonhole must contain more than one item”
  • 例如我们将read均分成三个不重叠的块，那么就可在reads错配不超过两个的情况下。至少有一个片段可以完全作为匹配的种子来帮助查询索引，大大提高了reads mapping的速度
  • After splitting the read into n(non-overlapped) blocks,there will be at least n-m perfectly-matched blocks(i.e. withou any mismatch with in the block) by following up-to-m mismatches
  • 于是，像ELAND MAQ SOAP1均广泛地运用了这一思路
  • 但是这也导致了新能的极具下降

Prefix Tree and Suffix Tree


数据压缩算法中的前缀树和后缀树开始被应用于reads mapping，这些数据结构，通过合并和共享子串，降低内存消耗，并且可以方便地实现对于最长公共子串的查找

  • 例如目前流行的的一些方式都采用了 Burrows-Wheeler transform(BTW)
    • BOWTIE
    • BWA
    • SOAP3
      可以逐位对比

当通过索引查找对应的主序列之后，我们就可以参照之前BLAST的思路，向左右两个方向延伸扩展之后，利用动态规划算法来确定最后的比对

但是我们需要注意，目前新一代测序计算的Quality不够高

Mapping Quality

假设我们看到的错配都是测序错误引起的
Given reference sequence z(length L),a read sequence x(length l),u is the alignment position of x on z,the probability that z actually coming from the position u is $p(z|x,u)$
$p(x|x,u)=\prod \limits_{mismatch}p(z_i)$
$SQ(u)=log(p(z|x,u))=\sum \limits_{mismatch} p(z_i)=\sum \limits_{mismatch} Q(z_i)$

If we assume that uniform NULL model,i.e. the read can randomly come from all possible positions with equal probability,then the error of mapping to a specified position u could be written as
$E(u)=\frac{SQ(u)}{\sum_iSQ(i)}$

Genetic Variants

  • SNV:Single Nucleotide Variant(最常设计到的遗传变异)
    • Substitution(SNP)
    • Indel:isertion/deletion
  • Structural Variation(SV)
    • Large-scale insertion/deletion
    • Inversion
    • Translocation
    • Copy Number Vairation(CNV)

SNV Calling is NOT Genotyping（not 基因型？）

  • “SNP calling aims to determine in which positions there are polymorphisms or in which positions at least one of the bases differs from a reference sequence”
  • “Genotype calling is the process of determing the genotype for each individual and is typically only done for positions in which a SNP or a ‘variant’ has already been called.”
  • 基因型：是指某一生物个体全部基因组合的总称。它反应生物体的遗传构成，即双亲获得的全部基因的总和。遗传学中具体使用的基因型，往往指某一形状的基因型。两个生物只要有一个基因型不同，那么它们的基因型就不相同。

Couting:an intuitive(and naive) approach

  • counting high-confident,non-reference allele(i.e. Quality>=20)
    • Freq<20% or >80% : homozygous genotype
    • Otherwise: heterozygous
  • Works well for “deep sequenced region” (DSR) ,i.e. depth>25x
    • But suffer from uder-calling of heterozygous genotypes for low-coverage regions
    • And can’t give an objective measurement for reliability

A simple Probabilistic Model for Genotyping

  • For diploid genome,there will be at most two different alleles(A and a) observed at a given site:
    • 3 possible genotypes:<A,A>,<A,a>,<a,a>
    • Number of A:k;Number of a:n-k
  • Then the probability for each genotypes is
    • P(D|<A,A>) = the probability that we have (n-k) sequencing errors at this site $\prod_{n-k} P(x_i)$
    • Similarly.we can see the P(D|<a,a>) = $\prod_{k}P(x_i)$
    • $P(D|<A,a>)=1-(P(D|<A,A>)+P(D|<a,a>))$
  • Bayes Formula can be further employed to calculate posterior probabilities,i.e. $P(<A,A>|D),P(<A,a>|D),P(<a,a>|D)$ if we can estimate the prior probabilities $P(<A,A>),P(<A,a>),P(<a,a>)$
  • 但是真实考虑的模型要考略的问题更多：Genome Analysis ToolKit(GATK)
