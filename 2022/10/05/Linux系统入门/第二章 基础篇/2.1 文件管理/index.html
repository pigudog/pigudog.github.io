Linux-基础篇-2.1 文件管理

Created2022-10-05|Updated2022-11-11|computerLinux
|Post View:

我们学习的核心就是建立在linux kernel上的shell
我们在桌面右键打开的terminal是一个仿真包，开机启动时的大黑屏才是真正的Linux shell

我们可以按住ctrl+Alt+f2-f6进入真正的终端控制台
按住ctrl+alt呼唤出鼠标
ctrl+alt+f1退出终端

root@pigudogzyy001 ~

  • root用户
  • pigudogzyy001,装系统时设置的机器名字
  • ~ 指的是当前的目录
  • 普通用户使用“$”

文件管理

Linux目录结构简介

1. windows：以多根的方式组织文件C:\ D:\  E:\
2. Linux:以单根的方式组织文件/ :-_ / \ |

注意：Linux运用正斜杠可以很好的避免windows里的 \ \ 的输入方式

  • cd:change directory
  • pwd:print work direcctory
  • ls:list files

根

系统管理员：root根
系统的目录（文件夹）：”根”：/
这两个玩意是有区别的

当然在文件角度：我们就可以从一级目录-二级目录-三级目录

Linux目录功能


对于Linux的储存配置的逻辑关系和真实的关系是不一样的，又称为虚拟目录–>挂载点

切换到根目录下查看：

  • bin 普通用户使用的命令：/bin/ls,/bin/data；Binary的缩写，存放着可以直接可以执行的常用命令根目录下的bin其实只是一个链接，实质是/usr/bin
  • sbin管理员使用的命令：/sbin/serve；s是system的意思，系统级的二进制命令，系统管理员或超级用户可以使用的系统管理工具。usr/sbin
  • lib:library系统文件 == windoes/System32，usr/lib
  • lib64：关于64位操作系统的一些特殊文件,usr/lib64
  • dev设备文件/dev/sda,/dev/sda1，所有的硬件设备，cpu，disk等都有相应的映射
  • root -root用户的HOME-“家长”的卧室
  • home-存储普通用户家目录-学习视频-狗头-是不会被别人看到的-不能随意访问的文件
  • etc-唯一不用记的，经常用/etc etc不是什么缩写，是and so on的意思 来源于 法语的 et cetera，所有的命令按钮文件的存储位置
  • run-运行文件是临时的
  • usr-like Program Files(x86)非常的重要，用户级别的东西
  • var-variable，经常被修改的可变化的东西，像log日志
  • opt-option,我们的第三方软件包安装的位置
  • media-可以识别一些可移动媒体设备，u盘，光驱
  • mnt-移动化设备的另一个挂载点
  • proc-process存放系统进程的信息，你不要动就好了
  • srv-service，跟系统服务相关的东西
  • sys-system，存放系统硬件信息
  • tmp-template，临时存放的位置，清空来节省空间

文件命令的初步理解

cd:绝对路径或者相对路径
ls:列出当前目录的内容
pwd：打印当前目录

命令（注意空格） 选项 参数
ls / 默认显示根目录的内容
ls -l / 长格式显示根下的详细内容
