001 Shell的概述

大数据程序员为什么要学习shell呢？

  • 看懂运维人员编写的shell程序
  • 偶尔会编写一些简单的shell程序来管理集群，提高开发效率

对于生信分析来说

  • 这次shell编程的学习是在进行完rna-seq之后的一次查漏补缺
  • 可以让我们对Linux的操作又更加深刻的理解

包在内核的外壳，是底层与外部应用的桥梁

  • Shell就是一个命令行解释器，它接受应用程序/用户命令，然后调用操作系统内核
  • Shell还是一个功能相当强大的编程语言，易编写、易调试、灵活性强
  • 但现在有一个强大的对手：python

当我们把多行命令放在一个文本文件中进行执行，那这玩意就叫一个脚本文件

所以接下来我们就要把shell当成一个功能相当强大的编程语言，进行脚本的编写

直接执行

系统环境变量里面有一个PATH的东西
在该目录下的脚本可以直接执行

  • PATH 是linux一个重要的环境变量，那么我们应该如何向该变量添加目录呢？
  • 路径环境变量的作用
    • 环境变量控制shell的行为。如果曾经在linux上使用过命令行，系统将以来PATH变量来查找正在输入的命令的位置
    • PATH 是 Linux 中的一个内置环境变量，它告诉 shell 在哪些目录中搜索可执行文件以响应用户或应用程序发出的命令。 它包含以冒号分隔的目录列表，用于查找您输入的命令
  • 那么如何在linux中将目录添加到PATH上呢？
    • 搜索 PATH 的顺序也很重要。因此，如果您有两个共享相同名称的可执行文件位于两个不同的目录中，则 shell 将运行位于 PATH 中第一个目录中的文件。
    • 临时添加目录到PATH
      • 现在假设您已经创建了一个位于/srv/scripts目录中的自定义 shell 脚本，并且您希望将此目录添加到您的 PATH 变量中。这是你应该做的：export PATH=$PATH:/srv/scripts
        • export 命令会将修改后的 PATH 变量导入到shell子进程环境中
      • 此外，如果您认为您的目录应该在其他所有内容之前被搜索，您可以将其添加到$PATH ：export PATH=/srv/scripts:$PATH
      • 请务必注意，如果您退出终端或从系统注销，PATH 将恢复并且更改将丢失，因为这种设置 PATH 的方法仅在当前终端会话期间临时记住更改。
    • 永久添加目录
      • 要使更改永久生效，您需要在 shell 配置文件中定义 PATH 变量。
      • 默认系统范围的 PATH 值在 /etc/profile 文件中指定。将目录添加到单个用户的路径的最佳位置是修改该用户的.bashrc文件。
        • vim ~/.bashrc
        • `export PATH=$PATH:/srv/scripts
        • source ~/.bashrc
        • echo $PATH 最后确认一下即可

009 shell工具

cut

cut 命令

  • 从文件的每一行剪切字节，字符和字段并将这些字节，字符字段输出
  • cut $[选项参数]$ filename
    • -f 列号，提取第几列
    • -d 分隔号，按照制定分隔符分割列，缺省为制表符

先准备一下数据

sed

sed

  • 是一种流编辑器，它一次处理一行内容，处理时，把当前把当前处理的行存储在临时缓冲区中，成为“模式空间”,接着sed命令处理缓冲区中的内容，处理完成后，把缓冲区的内容送往屏幕，接着处理下一行，这样不断重复，直到文件末尾。
  • 文件内容并没有改变，除非你使用重定向存储输出
  • sed $[选项参数]$ ‘command’ filename
    • -e 直接在指令列模式上进行sed的动作编辑
    • 命令功能描述
      • a 新增
      • d 删除
      • s 查找并替换

awk

awk

  • 一个强大文件分析工具，把文件逐行的读入，以空格为默认分隔符将每行切片，切开的部分再进行分析处理
  • awk $[选项参数]$ ‘pattern1 {action1} pattern2{action2}..’ filename
    • -F 指定输入文件分隔符
    • -v 赋值一个用户定义变量

sort

  • sort 命令是在Linux里非常有用，它将文件进行排序，并将排序结果标准输出
    • sort (options) (arguments)
      • -n 依照数值大小排列
      • -r 以相反的顺序排列
      • -t 排列时使用的分隔符
      • -k 制定需要排列的列
